ViMSUIT Results Repository

This repository contains the intermediate and final result artifacts generated by ViMSUIT, a vision–language–driven framework for automated mobile UI test case and test script generation from application screenshots.

Repository Structure
ViMSUIT/
├── Stage 1 Results/
│   ├── descriptions_txt/
│   ├── testcases_csv/
│   └── summary_top200_Screens.csv
│
├── Stage 2 Results/
│   ├── Semantic_Analysis/
│   ├── summary/
│   ├── test_scripts/
│   ├── consolidated_summary_<timestamp>.csv
│   ├── TestCase_Generation_Single_Image.*
│   └── TestCase_Generation_With_200_Screens.*

Stage 1 Results

Stage 1 contains outputs related to screen-level UI understanding and test case generation.

descriptions_txt/: Textual UI descriptions generated from screenshots, one file per screen ID.

testcases_csv/: Structured test cases generated for each screen.

summary_top200_Screens.csv: Metadata summary for the selected top-200 ENRICO screenshots.

Stage 2 Results

Stage 2 contains semantic analysis, validation outputs, and executable test artifacts.

Semantic_Analysis/: JSON files capturing structured semantic interpretation per screen.

summary/: Intermediate summary files from Stage 2 processing.

test_scripts/: Generated executable test scripts.

consolidated_summary_<timestamp>.csv: Aggregated results across all evaluated screens.

TestCase_Generation_Single_Image: Script for single-image test generation.

TestCase_Generation_With_200_Screens: Script for batch processing of multiple screenshots.

Notes

File names are indexed by ENRICO screen IDs.

Stage 1 outputs serve as inputs to Stage 2.

The structure supports transparent inspection and reproducibility of all experimental results.
